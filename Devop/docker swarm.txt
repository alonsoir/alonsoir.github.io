Good afternoon Xavier,

I am writing to propose that we investigate the possibility of using docker swarm instead of docker compose to be able to deploy the software to our clients, and the reason is to guarantee the resilience of the stored processes in the containers, that is, if a container is left In an unhealthy state, docker-compose doesn't have all the features to try to remount the container. This state can be for multiple reasons, as we have found in our team today the hard way, in DEV3 we had a lot of unhealthy services and docker compose was not able to raise them. We had to stop them all and pick them up one by one. Docker Swarm and k8s provide finer control over survivability, giving customers greater capability. I suggest that you give us permission to explore docker swarm, I already know that other colleagues are with k8s.
